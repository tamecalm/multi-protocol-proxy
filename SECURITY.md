# Security Policy

We take the security and privacy of our users very seriously. Since the Multi-Protocol Proxy routes network traffic, maintaining high-security standards is our top priority.

---

## Supported Versions

Only the latest release version on the `main` branch is actively supported with security updates.

| Version | Supported |
| ------- | --------- |
| Latest Release | ✅ Yes |
| < Latest Release | ❌ No |

## Reporting a Vulnerability

**Please do not report security vulnerabilities through public GitHub issues.**

If you discover a security vulnerability, please report it directly to the maintainer:
- Reach out privately via Twitter/X direct message to [@tamecalm](https://twitter.com/tamecalm).
- We will acknowledge receipt of your report within 48 hours and provide updates on resolution.

Please include:
- A description of the vulnerability.
- Steps to reproduce or a proof of concept (PoC).
- Potential impact (e.g. bypass authentication, remote code execution, crash/DoS).

We follow responsible disclosure practices. We ask that you give us reasonable time to investigate and patch the issue before making details public.
